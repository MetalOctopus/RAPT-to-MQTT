RAPT2MQTT

Bridge between RAPT (Kegland) brewing devices and MQTT for Home Assistant and other home automation platforms.

The bridge polls the RAPT cloud API for temperature controller data and publishes it to your MQTT broker. It also listens for TILT hydrometer data on MQTT.

Getting Started

  1. Go to Config and enter your MQTT broker address and port.
  2. Enter your RAPT email and API Secret. You can create an API secret in the RAPT Portal under My Account > API Secrets.
  3. Set the Poll Interval (default 300s). The RAPT API has no official rate limit, but polling faster than 60s is not recommended.
  4. Click Save Configuration, then Start the bridge.
  5. Your RAPT devices will appear in the sidebar as they are discovered.

MQTT Topics

RAPT Data RAPT/temperatureController
RAPT Commands RAPT/temperatureController/Command
TILT Data (inbound) TiltPi

RAPT Payload Example

{
  "device_id": "abc-123",
  "name": "Fermenter",
  "current_temp": "18.50",
  "target_temp": "18.00",
  "cooling_enabled": true,
  "heating_enabled": false,
  "connection_state": "Connected",
  "rssi": -65
}

Set Temperature Command

Publish to RAPT/temperatureController/Command:

"Temperature" : "20.00"

TILT Hydrometer Integration

RAPT2MQTT can display live data from a TILT hydrometer if it publishes to your MQTT broker. The bridge listens on the TiltPi topic.

What is TILT?

The TILT is a free-floating Bluetooth hydrometer that measures specific gravity and temperature inside your fermenter. It broadcasts via BLE (no WiFi), so it needs a bridge device to reach your network.

Option 1: TiltPi + Custom MQTT Node (Recommended if you already have a TiltPi)

TiltPi is a Raspberry Pi running Node-RED that reads TILT BLE data. It does not publish to MQTT by default, but adding it is straightforward:

  1. Open your TiltPi Node-RED editor at http://tiltpi.local:1880
  2. Find the ui_template node for your TILT color (the display card that shows temperature and SG)
  3. Add a function node after it with this code: 
    msg.topic = "TiltPi";
msg.payload = JSON.stringify({
  major: parseFloat(msg.payload.displayTemp || 0),
  minor: Math.round((parseFloat(msg.payload.SG) || 0) * 1000)
});
return msg;
  4. Optionally add an rbe (Report By Exception) node after the function to only send on changes
  5. Add an mqtt out node. Configure it with:
    • Server: Your MQTT broker address (same one configured in RAPT2MQTT)
    • Topic: TiltPi
    • QoS: 0
  6. Wire: ui_template → function → rbe → mqtt out
  7. Click Deploy

Option 2: TiltBridge (ESP32)

If you don't have a TiltPi, a TiltBridge is a cheap (~$10) ESP32 board that scans for TILT BLE and publishes to MQTT natively.

  1. Flash TiltBridge firmware onto an ESP32
  2. Configure it to publish to your MQTT broker
  3. TiltBridge publishes to tiltbridge/tilt_<Color> by default — you may need to adapt the topic or add a Node-RED rule to reformat for RAPT2MQTT

Payload Format

RAPT2MQTT expects this JSON on the TiltPi topic:

{
  "major": 68.0,
  "minor": 1050
}
majorTemperature (calibrated, typically °F)
minorSpecific Gravity × 1000 (e.g. 1050 = SG 1.050)

TILT Colors

TILTs come in 8 colors, each with a unique Bluetooth UUID. If you have multiple TILTs, you'll need to modify the Node-RED flow to include the color in the topic or payload.
