RAPT2MQTT

Bridge between RAPT (Kegland) brewing devices and MQTT for Home Assistant and other home automation platforms.

The bridge polls the RAPT cloud API for temperature controller data and publishes it to your MQTT broker. It also listens for TILT hydrometer data on MQTT.

Getting Started

  1. Go to Config and enter your MQTT broker address and port.
  2. Enter your RAPT email and API Secret. Create one in the RAPT Portal under My Account > API Secrets.
  3. Set the Poll Interval (default 300s). Polling faster than 60s is not recommended.
  4. Click Save Configuration, then Start the bridge.
  5. Devices appear in the sidebar as they are discovered.

TILT Hydrometer Integration

RAPT2MQTT displays live TILT data if it publishes to your MQTT broker on the TiltPi topic.

TiltPi + Custom MQTT Node

  1. Open TiltPi Node-RED at http://tiltpi.local:1880
  2. Find the ui_template node for your TILT color
  3. Add a function node with: 
    msg.topic = "TiltPi";
var p = msg.payload;
msg.payload = JSON.stringify({
  color: (p.Color || "Unknown"),
  beer: (p.Beer && p.Beer[0]) || "",
  temperature: parseFloat(p.displayTemp || 0),
  temperature_raw: parseFloat(p.Temp || 0),
  sg: parseFloat(p.SG || 0),
  rssi: Math.round((parseInt(p.rssi || 0)) / 5) * 5,
  mac: p.mac || "",
  uuid: p.uuid || ""
});
return msg;
  4. Add rbe node, then mqtt out node pointed at your MQTT broker, topic TiltPi
  5. Wire: ui_template → function → rbe → mqtt out and Deploy

Alternative: TiltBridge (ESP32, ~$10)

Flash TiltBridge firmware onto an ESP32 for native MQTT output.
